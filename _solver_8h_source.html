5  \brief Charm++ chare linear system merger group to solve a linear system
6  \details Charm++ chare linear system merger group used to collect and
7  assemble the left hand side matrix (lhs), the right hand side (rhs) vector,
8  and the solution (unknown) vector from individual worker
9  chares. Beside collection and assembly, the system is also solved. The
10  solution is outsourced to hypre, an MPI-only library. Once the solution is
11  available, the individual worker chares are updated with the new solution.
12 
13  This class assembles and solves two linear systems, whose rhs vectors may
14  change during time stepping. One of the two linear systems is called a
15  high-order and the other one is the low-order linear system.
16 
17  Characteristics of the low-order linear system: (1) the left hand side
18  matrix is diagonal, (2) the right hand side vector is a combination of the
19  high-order right hand side vector and another vector, assembled separately
20  (and overlapped with that of the high-order system). This dual system
21  solution is done here as required by the flux-corrected transport algorithm
22  used for transport equations in inciter.
23 
24  The implementation uses the Charm++ runtime system and is fully
25  asynchronous, overlapping computation and communication. The algorithm
26  utilizes the structured dagger (SDAG) Charm++ functionality. The high-level
27  overview of the algorithm structure and how it interfaces with Charm++ is
28  discussed in the Charm++ interface file src/LinSys/solver.ci.

323  //! Receive lower and upper global node IDs all PEs will operate on
324  //! \param[in] p PE whose bounds being received
325  //! \param[in] lower Lower index of the global rows on my PE
326  //! \param[in] upper Upper index of the global rows on my PE
327  void bounds( int p, std::size_t lower, std::size_t upper ) {
328  Assert( lower < upper, "Lower bound must be lower than the upper bound: "
329  "(" + std::to_string(lower) + "..." + std::to_string(upper) +
330  ") sent by PE " + std::to_string(p) );
331  // Store our bounds
332  if (p == CkMyPe()) {
333  m_lower = lower;
334  m_upper = upper;
335  }
336  // Store inverse of PE-division map stored on all PE
337  m_div[ {lower,upper} ] = p;
338  // If we have all PEs' bounds, signal the runtime system to continue
339  if (m_div.size() == static_cast<std::size_t>(CkNumPes())) {
340  // Create my PE's lhs matrix distributed across all PEs
341  m_A.create( m_lower*m_ncomp, m_upper*m_ncomp );
342  // Create my PE's rhs and unknown vectors distributed across all PEs
343  m_b.create( m_lower*m_ncomp, m_upper*m_ncomp );
344  m_x.create( m_lower*m_ncomp, m_upper*m_ncomp );
345  // Create linear solver
346  m_solver.create();
347  // Signal back to host that setup of workers can start
348  Group::contribute( m_cb.get< tag::coord >() );
349  }

383  //! Chares contribute their global row ids
384  //! \param[in] fromch Charm chare array index contribution coming from

389  // Collect global ids of workers on my PE
390  m_myworker.push_back( fromch );
391  // Store rows owned and pack those to be exported, also build import map
392  // used to test for completion
393  std::map< int, std::set< std::size_t > > exp;
394  for (auto gid : row) {
395  if (gid >= m_lower && gid < m_upper) { // if own
396  m_rowimport[ fromch ].push_back( gid );
397  m_row.insert( gid );
398  } else exp[ pe(gid) ].insert( gid );
399  }
400  // Export non-owned parts to fellow branches that own them
401  m_nperow += exp.size();
402  for (const auto& p : exp) {
403  auto tope = static_cast< int >( p.first );
404  Group::thisProxy[ tope ].addrow( fromch, CkMyPe(), p.second );

406  checkifrowcomplete();
