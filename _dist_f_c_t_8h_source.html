2 /*!

4  \copyright 2012-2015, J. Bakosi, 2016-2018, Los Alamos National Security, LLC.
5  \brief Charm++ module interface for distributed flux-corrected transport
6  \details Charm++ module interface file for asynchronous distributed
7  flux-corrected transport (FCT).
8 
9  There are a potentially large number of DistFCT Charm++ chares created by
10  Transporter. Each DistFCT gets a chunk of the full load (part of the mesh)
11  and performs flux-corrected transport.
12 
13  The implementation uses the Charm++ runtime system and is fully
14  asynchronous, overlapping computation and communication. The algorithm
15  utilizes the structured dagger (SDAG) Charm++ functionality. The high-level
16  overview of the algorithm structure and how it interfaces with Charm++ is
17  discussed in the Charm++ interface file src/Inciter/distfct.ci.
18

20  The following is a directed acyclic graph (DAG) that outlines the
21  asynchronous algorithm implemented in this class The detailed discussion of
22  the algorithm is given in the Charm++ interface file transporter.ci, which
23  also repeats the graph below using ASCII graphics. On the DAG orange
24  fills denote global synchronization points that contain or eventually lead
25  to global reductions. Dashed lines are potential shortcuts that allow
26  jumping over some of the task-graph under some circumstances or optional
27  code paths (taken, e.g., only in DEBUG mode). See the detailed discussion in
28  distfct.ci.
