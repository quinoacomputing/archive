7  \details Advanced Charm++ array creation refers to various ways arrays can

9  http://charm.cs.illinois.edu/manuals/html/charm++/manual.html, Sec.
10  Advanced Array Creation. This class does a distribution that is based on
11  which portion of a distributed sparse matrix resulting from discretization
12  on an unstructured grid residing on a PE should hold a given chare array
13  element. (The one that owns most on-PE rows to minimize off-PE
14  communication.)
15 
16  Note that to help with performance, it is not advised to do heavy
17  computations in the overridden member functions, procNum() and
18  populateInitial(), since they can be potentially called many times. See
19  also this note on the inner workings of the Charm++ runtime system
20  regarding map objects and array element creation from long-time Charm++
21  developer, Eric Bohm, also at:
22  http://lists.cs.uiuc.edu/pipermail/charm/2015-May/002047.html
23 
24  _Procnum will be called to construct your chare, the first time a message
25  is sent to it from a node, and each time subsequent sends do not find a
26  precached location record for it. The latter event can occur when many
27  sends have pushed it out of cache, or after migration._
28 
29  _The potential global memory footprint of location management caching is
30  proportional to the total number of objects multiplied by the number of
31  nodes. Therefore, the runtime system keeps a finite number on each node.
32  At the limit, procnum could be called for nearly every message send,
33  therefore procnum should be designed to be inexpensive._
34 
35  The heavy portion of array element placement should therefore be done in
36  the constructor.
37 */
