49  //! \brief Constructor: set all defaults.
50  //! \param[in] ctrinfo std::map of control file keywords and their info
51  //! \details Anything not set here is initialized by the compiler using the
52  //! default constructor for the corresponding type. The ctrinfo map
53  //! argument is optional. If not given, it is an empty std::map
54  //! constructed in-place and affects nothing. If given, it contains the
55  //! control file keywords, all of which are moved into the relevant slot
56  //! (tag::ctrinfo). This allows constructing, e.g., a CmdLine object both
57  //! with and without this information in place, which are both used at
58  //! different stages of the execution. For example, because the
59  //! command-line is parsed very early on during runtime while the input
60  //! deck is only parsed much later, the control-file keywords and their
61  //! information (owned by and generated by the input deck and its
62  //! constructor) is not yet available when the CmdLine object is
63  //! constructed. However, during command-line parsing it is still possible
64  //! to request information on a control file keyword, so it must be
65  //! available. The input deck is where all parsed information goes during
66  //! control file parsing and is stored at global scope, see, e.g.,
67  //! walker::g_inputdeck. This global-scope (still namespace-scope), input
68  //! deck object is thus created before command-line parsing. The input
69  //! deck object's constructor (working only on type information, available
70  //! at compile-time, of all the control file keywords) creates a run-time
71  //! map. This is a run-time map, but available before main() starts,
72  //! because it is const and it is initialized as a global-scope map. This
73  //! map is then passed in here as ctrinfo, and its contents inserted into
74  //! the CmdLine object, making the control-file keywords and their info
75  //! available during command-line parsing. Since the input deck stack
76  //! contains a copy of the command-line stack, the command-line stack must
77  //! be possible to be instantiated without passing the ctrinfo map,
78  //! otherwise it would be a mutual dependency.
79  CmdLine( tk::ctr::HelpFactory ctrinfo = tk::ctr::HelpFactory() ) {
80  set< tag::verbose >( false ); // Use quiet output by default
81  // Initialize help: fill from own keywords + add map passed in
82  boost::mpl::for_each< keywords >( tk::ctr::Info( get< tag::cmdinfo >() ) );
83  get< tag::ctrinfo >() = std::move( ctrinfo );
84  }
