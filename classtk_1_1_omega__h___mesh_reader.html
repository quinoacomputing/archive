Detailed Description

Omega_h mesh-based data reader

Mesh reader class facilitating reading from mesh-based field data a file in Omega_h format.

See also
https://github.com/ibaned/omega_h

Read part of the mesh (graph and coords) from Omega_h file

Total number of PEs defaults to 1 for a single-CPU read, this PE defaults to 0 for a single-CPU read.

Parameters
[in,out]ginpoelContainer to store element connectivity of this PE's chunk of the mesh (global ids)
[in,out]inpoelContainer to store element connectivity with local node IDs of this PE's mesh chunk
[in,out]triinpContainer to store triangle element connectivity (if exists in file) with global node indices
[in,out]gidContainer to store global node IDs of elements of this PE's mesh chunk
[in,out]lidContainer to store global->local node IDs of elements of this PE's mesh chunk
[in,out]coordContainer to store coordinates of mesh nodes of this PE's mesh chunk
[in]numpesTotal number of PEs (default n = 1, for a single-CPU read)
[in]mypeThis PE (default m = 0, for a single-CPU read)
Note
The last two integer arguments are unused. They are needed because this function can be used via a polymorphic interface via a base class, see tk::MeshReader, and other specialized mesh readers, e.g., tk::ExodusIIMeshReader, use these arguments. Here we require the Omega_h input mesh to be pre-partitioned, with Omega_h's osh_part tool, into the number of partitions that is equal to the number of PEs this fuinction is called on in parallel.
