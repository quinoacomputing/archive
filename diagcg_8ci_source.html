51  // Interpretation of the above DAG
52  // -------------------------------
53  // Control flow is from left to right and top to bottom.
54  //
55  // There are two parts of the DAG below: (1) setup: tasks leading to the
56  // start of time stepping, and (2) solve: tasks leading to solving the
57  // high and low order linear systems. Setup only happens once, while solve
58  // happens every time step.
59  //
60  // (1) Setup: The left hand side matrix (lumped-mass matrix, where only
61  // the diagonals are stored in a vector) are computed and assembled on the
62  // chare boundaries. Only when own contributions are complete and when
63  // chare-boundary contributions are received can timestepping start. This
64  // happens independently on each chare, communicating with their
65  // respective neighbors. Once the left hand side (diagonal) matrix is
66  // assembled it is assumed to never change during time stepping.
67  //
68  // (2) Solve: The right hand side vector contributions are computed and
69  // assembled on the chare boundaries. For the low order solution there is
70  // an additional, mass-diffusion, term added to the right hand side. Only
71  // when own contributions to both terms on the right hand side are
72  // complete and when chare-boundary contributions to both terms on the
73  // right hand side are received can the two systems be solved. This
74  // happens independently on each chare, communicating with their
75  // respective neighbors.
76
