2 /*!
3  \file src/Base/Variant.h
4  \copyright 2012-2015, J. Bakosi, 2016-2017, Los Alamos National Security, LLC.
5  \brief Variant is a wrapper around boost::variant to make it PUPable
6  \details Variant is a wrapper around boost::variant to make it PUPable.
7 
8  Since boost::variant (as well as std::variant) when default-constructed is
9  initialized to hold a value of the first alternative of its type list,
10  calling PUP that works based on a boost::visitor with a templated operator()
11  always incorrectly triggers the overload for the first type. Thus when
12  PUPing a variant not only its value but its type must also be sent during
13  migration. The class below achieves this by reading out not only the value
14  but also its zero-based index of the type alternative that is currently held
15  by the variant passed to its initializer constructor. The index and the
16  value are stored as an integer and a value in a tuple that can hold a value
17  of any of the alternative types the variant can. The index and tuple are
18  then PUPed and when unpacking, as an additional step, the variant is
19  reconstructed using the index and the value in the tuple. This latter is
20  done by invoking a runtime loop across all alternative types using
21  boost::mpl::for_each. The final variant can then be accessed by the get()
22  public member function.
23 
24  \see UnitTest/tests/Base/TestPUPUtil.h or Inciter::SchemeBase.h for
25  tk::Variant in action.
26 
27  \note Type-based access to tuple elements as in std::get<T>( tuple ) requires
28  C++14.
29 */
