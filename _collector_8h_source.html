32 //! Collector Charm++ chare group class
33 //! \details Instantiations of Collector comprise a processor aware Charm++
34 //! chare group. When instantiated, a new object is created on each PE and not
35 //! more (as opposed to individual chares or chare array object elements). The
36 //! group's elements are used to collect information from all Integrator chare
37 //! objects that happen to be on a given PE. See also the Charm++ interface
38 //! file collector.ci.
39 //! \see http://charm.cs.illinois.edu/manuals/html/charm++/manual.html
