Detailed Description

template<class Physics, class Problem>
class inciter::dg::CompFlow< Physics, Problem >

CompFlow used polymorphically with tk::DGPDE.

The template arguments specify policies and are used to configure the behavior of the class. The policies are:

  • Physics - physics configuration, see PDE/CompFlow/Physics.h
  • Problem - problem configuration, see PDE/CompFlow/Problems.h
    Note
    The default physics is Euler, set in inciter::deck::check_compflow()

◆ RiemannFactory

template<class Physics , class Problem >
using inciter::dg::CompFlow< Physics, Problem >::RiemannFactory = std::map< ctr::FluxType, std::function< RiemannSolver() > >
private

Factory for Riemann solvers

This factory is used to store the constructors as a std::function of specific Riemann solvers that can be invoked at a later time compared to the point where the map is populated. The key is an enum, uniquely idenfitying a specific Riemann solver. The value is std::function storing a constructor to be invoked. The type of object stored in std::function is a generic (base) class constructor, which provides a polymorphyic interface (overridable functions) that specific (child) Riemann solvers override.
