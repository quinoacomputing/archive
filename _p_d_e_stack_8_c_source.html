36 // *****************************************************************************
37 // Constructor: register all partial differential equations into factory
38 //! \details This constructor consists of several blocks, each registering a
39 //! potentially large number of entries in the partial differential equation
40 //! factory, m_factory, which is of type inciter::PDEFactory, a std::map. At
41 //! this time, each type of partial differential equation can be configured to
42 //! use a unique _problem policy_. (More types of policies will most likely
43 //! come in the future.) Policy classes are template arguments to the partial
44 //! differential equation classes and influence their behavior in a different
45 //! way, abstracting away certain functions, e.g., how to set problem-specific
46 //! initial and/or boundary conditions and how to update their coefficients
47 //! during time integration. For more information on policy-based design, see
48 //! http://en.wikipedia.org/wiki/Policy-based_design. This abstraction allows
49 //! [separation of concerns](http://en.wikipedia.org/wiki/Separation_of_concerns).
50 //!
51 //! Since the functionality of the policies are orthogonal to each other,
52 //! i.e., they do not depend on each other or their host (the partial
53 //! differential equation class), a Cartesian product of combinations are
54 //! possible, depending on which policies are selected. _This constructor
55 //! registers all possible combinations of policies for all available
56 //! differential equations._ By _register_, we mean, an entry is recorded in
57 //! an associative container, a std::map, that associates a lightweight key of
58 //! type inciter::ctr::PDEKey, consisting of only an enum for each policy
59 //! type, to an std::function object that holds the constructor bound to its
60 //! arguments corresponding to a particular partial differential equation +
61 //! policies combination. Note that registering these entries in the map does
62 //! not invoke the constructors. The mapped value simply stores how the
63 //! constructors should be invoked at a later time. At some point later,
64 //! based on user input, we then instantiate only the partial differential
65 //! equations (and only those configurations) that are requested by the user.
66 //!
67 //! Since all partial differential equation types (registered in the factory)
68 //! "inherit" from a common "base", client-code is unform and generic, and
69 //! thus immune to changes in the inner workings of the particular partial
70 //! differential equations as long as they fullfill certain concepts, i.e.,
71 //! implement certain member functinos, enforced by the _common base_, PDE.
72 //! The words "inherit and "base" are quoted here, because the common base
73 //! does not use inheritance in the normal OOP sense and does not use
74 //! reference semantics, i.e., pointers, visible to client-code either. The
75 //! relationship is more of a _models a_-type, which simplifies client-code
76 //! and allows for the benfits of runtime inheritance with value-semantics
77 //! which is less error prone and easier to read. See more about the
78 //! _models-a_ relationship and its implementation in PDE/PDE.h.
79 //!
80 //! The design discussed above allows the registration, instantiation, and
81 //! use of the partial differential equations to be generic, which eliminates
82 //! a lot of boiler-plate code and makes client-code uniform.
83 //!
84 //! _Details of registration using mpl::cartesian_product:_
85 //!
86 //! The template argument to mpl::cartesian_product requires a sequence of
87 //! sequences of types. We use vector of vectors of types, listing all
88 //! possible policies. The constructor argument to mpl::cartesian_product is a
89 //! functor that is to be applied to all combinations. mpl::cartesian_product
90 //! will then create all possible combinations of these types and call the
91 //! user-supplied functor with each type of the created sequence as a template
92 //! parameter. The user-supplied functor here is registerPDE, which, i.e.,
93 //! its constructor call, needs a single template argument, a class templated
94 //! on policy classes. This is the partial differential equation class to be
95 //! configured by selecting policies and to be registered. The arguments to
96 //! registerPDE's constructor are the factory, the enum denoting the
97 //! differential equation type, and a reference to a variable of type
98 //! std::set< ctr::PDEType >, which is only used internally to PDEStack
99 //! for counting up the number of unique differential equation types
100 //! registered, used for diagnostics purposes.
101 //! \author J. Bakosi
102 // *****************************************************************************
103 {
104  namespace mpl = boost::mpl;
105 
106  // Transport PDE
107  // Construct vector of vectors for all possible policies for PDE
108  using TransportPolicies = mpl::vector< TransportPhysics, TransportProblems >;
109  // Register PDE for all combinations of policies
110  mpl::cartesian_product< TransportPolicies >(
111  registerPDE< Transport >( m_factory, ctr::PDEType::TRANSPORT, m_eqTypes ) );
112 
113  // Compressible flow system of PDEs
114  // Construct vector of vectors for all possible policies for PDE
115  using CompFlowPolicies = mpl::vector< CompFlowPhysics, CompFlowProblems >;
116  // Register PDE for all combinations of policies
117  mpl::cartesian_product< CompFlowPolicies >(
118  registerPDE< CompFlow >( m_factory, ctr::PDEType::COMPFLOW, m_eqTypes ) );
119 }

121 std::vector< inciter::PDE >
122 PDEStack::selected() const
123 // *****************************************************************************
124 // Instantiate all selected partial differential equations
125 //! \return std::vector of instantiated partial differential equation objects
126 //! \author J. Bakosi
127 // *****************************************************************************
128 {
129  std::map< ctr::PDEType, ncomp_t > cnt; // count PDEs per type
130  std::vector< PDE > pdes; // will store instantiated PDEs
131 
132  for (const auto& d : g_inputdeck.get< tag::selected, tag::pde >()) {
133  if (d == ctr::PDEType::TRANSPORT)
134  pdes.push_back( createPDE< tag::transport >( d, cnt ) );
135  else if (d == ctr::PDEType::COMPFLOW)
136  pdes.push_back( createPDE< tag::compflow >( d, cnt ) );
137  else Throw( "Can't find selected PDE" );
138  }
139 
140  return pdes;
141 }

169 // *****************************************************************************
170 // Return information on the transport PDE
171 //! \param[inout] cnt std::map of counters for all partial differential equation
172 //! types
173 //! \return vector of string pairs describing the PDE configuration
174 //! \author J. Bakosi
175 // *****************************************************************************
176 {
177  auto c = ++cnt[ ctr::PDEType::TRANSPORT ]; // count eqs
178  --c; // used to index vectors starting with 0
179 
180  std::vector< std::pair< std::string, std::string > > nfo;
181 
182  nfo.emplace_back( ctr::PDE().name( ctr::PDEType::TRANSPORT ), "" );
183  nfo.emplace_back( "problem", ctr::Problem().name(
184  g_inputdeck.get< tag::param, tag::transport, tag::problem >()[c] ) );
185  nfo.emplace_back( "start offset in unknowns array", std::to_string(
186  g_inputdeck.get< tag::component >().offset< tag::transport >(c) ) );
187  auto ncomp = g_inputdeck.get< tag::component >().get< tag::transport >()[c];
188  nfo.emplace_back( "number of components", std::to_string( ncomp ) );
189  const auto& diff =
190  g_inputdeck.get< tag::param, tag::transport, tag::diffusivity >();
191  if (diff.size() > c)
192  nfo.emplace_back( "coeff diffusivity [" + std::to_string( ncomp ) + "]",
193  parameters( diff[c] ) );
194  const auto& u0 = g_inputdeck.get< tag::param, tag::transport, tag::u0 >();
195  if (u0.size() > c)
196  nfo.emplace_back( "coeff u0 [" + std::to_string( ncomp ) + "]",
197  parameters( u0[c] ) );
198  const auto& lambda =
199  g_inputdeck.get< tag::param, tag::transport, tag::lambda >();
200  if (lambda.size() > c)
201  nfo.emplace_back( "coeff lambda [" + std::to_string( ncomp ) + "]",
202  parameters( lambda[c] ) );
203 
204  return nfo;
