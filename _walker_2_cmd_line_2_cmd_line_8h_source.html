51  //! \brief Constructor: set all defaults.
52  //! \param[in] ctrinfo std::map of control file keywords and their info
53  //! \details Anything not set here is initialized by the compiler using the
54  //! default constructor for the corresponding type. The ctrinfo map
55  //! argument is optional. If not given, it is an empty std::map
56  //! constructed in-place and affects nothing. If given, it contains the
57  //! control file keywords, all of which are moved into the relevant slot
58  //! (tag::ctrinfo). This allows constructing, e.g., a CmdLine object both
59  //! with and without this information in place, which are both used at
60  //! different stages of the execution. For example, because the
61  //! command-line is parsed very early on during runtime while the input
62  //! deck is only parsed much later, the control-file keywords and their
63  //! information (owned by and generated by the input deck and its
64  //! constructor) is not yet available when the CmdLine object is
65  //! constructed. However, during command-line parsing it is still possible
66  //! to request information on a control file keyword, so it must be
67  //! available. The input deck is where all parsed information goes during
68  //! control file parsing and is stored at global scope, see, e.g.,
69  //! walker::g_inputdeck. This global-scope (still namespace-scope), input
70  //! deck object is thus created before command-line parsing. The input
71  //! deck object's constructor (working only on type information, available
72  //! at compile-time, of all the control file keywords) creates a run-time
73  //! map. This is a run-time map, but available before main() starts,
74  //! because it is const and it is initialized as a global-scope map. This
75  //! map is then passed in here as ctrinfo, and its contents inserted into
76  //! the CmdLine object, making the control-file keywords and their info
77  //! available during command-line parsing. Since the input deck stack
78  //! contains a copy of the command-line stack, the command-line stack must
79  //! be possible to be instantiated without passing the ctrinfo map,
80  //! otherwise it would be a mutual dependency.
81  CmdLine( tk::ctr::HelpFactory ctrinfo = tk::ctr::HelpFactory() ) {
82  set< tag::io, tag::output >( "out" );
83  set< tag::io, tag::pdf >( "pdf" );
84  set< tag::io, tag::stat >( "stat.txt" );
85  set< tag::virtualization >( 0.0 );
86  set< tag::verbose >( false ); // Quiet output by default
87  // Initialize help: fill from own keywords + add map passed in
88  boost::mpl::for_each< keywords >( tk::ctr::Info( get< tag::cmdinfo >() ) );
89  get< tag::ctrinfo >() = std::move( ctrinfo );
90  }
