62  //! \brief Constructor: set all defaults.
63  //! \param[in] ctrinfo std::map of control file keywords and their info
64  //! \details Anything not set here is initialized by the compiler using the
65  //! default constructor for the corresponding type. The ctrinfo map
66  //! argument is optional. If not given, it is an empty std::map
67  //! constructed in-place and affects nothing. If given, it contains the
68  //! control file keywords, all of which are moved into the relevant slot
69  //! (tag::ctrinfo). This allows constructing, e.g., a CmdLine object both
70  //! with and without this information in place, which are both used at
71  //! different stages of the execution. For example, because the
72  //! command-line is parsed very early on during runtime while the input
73  //! deck is only parsed much later, the control-file keywords and their
74  //! information (owned by and generated by the input deck and its
75  //! constructor) is not yet available when the CmdLine object is
76  //! constructed. However, during command-line parsing it is still possible
77  //! to request information on a control file keyword, so it must be
78  //! available. The input deck is where all parsed information goes during
79  //! control file parsing and is stored at global scope, see, e.g.,
80  //! walker::g_inputdeck. This global-scope (still namespace-scope), input
81  //! deck object is thus created before command-line parsing. The input
82  //! deck object's constructor (working only on type information, available
83  //! at compile-time, of all the control file keywords) creates a run-time
84  //! map. This is a run-time map, but available before main() starts,
85  //! because it is const and it is initialized as a global-scope map. This
86  //! map is then passed in here as ctrinfo, and its contents inserted into
87  //! the CmdLine object, making the control-file keywords and their info
88  //! available during command-line parsing. Since the input deck stack
89  //! contains a copy of the command-line stack, the command-line stack must
90  //! be possible to be instantiated without passing the ctrinfo map,
91  //! otherwise it would be a mutual dependency.
92  CmdLine( tk::ctr::HelpFactory ctrinfo = tk::ctr::HelpFactory() ) {
93  set< tag::io, tag::output >( "out" );
94  set< tag::io, tag::diag >( "diag" );
95  set< tag::io, tag::part >( "track.h5part" );
96  set< tag::virtualization >( 0.0 );
97  set< tag::verbose >( false ); // Quiet output by default
98  set< tag::benchmark >( false ); // No benchmark mode by default
99  set< tag::feedback >( false ); // No detailed feedback by default
100  // Initialize help: fill from own keywords + add map passed in
101  boost::mpl::for_each< keywords >( tk::ctr::Info( get< tag::cmdinfo >() ) );
102  get< tag::ctrinfo >() = std::move( ctrinfo );
