46 //! \brief Generic definition of a keyword
47 //! \details A keyword is a struct that collects the information that makes up a
48 //! keyword. The requirement on the first template argument, Info, is that it
49 //! must define the name(), shortDescription(,) and longDescription() member
50 //! functions returning compile-time (static) std::strings. The
51 //! shortDescription() member function is used to return a short description
52 //! of what the keyword is used for, while the longDescription() member
53 //! function is used for a longer, e.g., a paragraph-long, description on
54 //! what the keyword can be used for and how it can and should be used. The
55 //! last template parameter is a pegtl string, a list of character constants,
56 //! specifying the case-sensitive characters that make up the keyword, which
57 //! is then matched by the parser. The keyword must be at least one character
58 //! long, but otherwise its length is only limited by the compiler's
59 //! recursion handling capability of variadic templates. While the name(),
60 //! shortDescription() and longDescription() member functions of Info are
61 //! required, there are also optional ones, such as
62 //! Info::exptect::description(), which, if defined, must also be static and
63 //! must return a std::string, describing the type the particular keyword
64 //! expects during parsing. This is optional since not every keyword expects
65 //! a value (or values) of a particular type. For example, the keyword 'end'
66 //! is simply used to close a block in the input file, and what follows does
67 //! not have a relationship to the keyword. A counterexample is is 'title',
68 //! which expects a double-quoted string immediately after the keyword
69 //! title'.
70 //! \see For example client-code and more detailed documentation on the possible
71 //! fields, see Control/Keywords.h.
72 //! \author J. Bakosi
73 template< typename Info, typename > struct keyword;
