46 //! \brief Generic definition of a keyword
47 //! \details A keyword is a struct that collects the information that makes up a
48 //! keyword. The requirement on the first template argument, Info, is that it
49 //! must define the name(), shortDescription(,) and longDescription() member
50 //! functions returning compile-time (static) std::strings. The
51 //! shortDescription() member function is used to return a short description
52 //! of what the keyword is used for, while the longDescription() member
53 //! function is used for a longer, e.g., a paragraph-long, description on
54 //! what the keyword can be used for and how it can and should be used. The
55 //! last template parameters are a list of character constants, specifying
56 //! the case-sensitive characters that make up the keyword, which is then
57 //! matched by the parser. The keyword must be at least one character long,
58 //! but otherwise its length is only limited by the compiler's recursion
59 //! handling capability of variadic templates. While the name(),
60 //! shortDescription() and longDescription() member functions of Info are
61 //! required, there are also optional ones, such as
62 //! Info::exptect::description(), which, if defined, must also be static and
63 //! return a std::string, describing the type the particular keyword expects
64 //! during parsing. This is optional since not every keyword expects a value
65 //! (or values) of a particular type. For example, the keyword 'end' is
66 //! simply used to close a block, and what follows does not have a
67 //! relationship to the keyword. A counterexample is is 'title', which
68 //! expects a double-quoted string.
69 //! \see For example client-code and more detailed documentation on the possible
70 //! fields, see Control/Keywords.h.
71 //! \author J. Bakosi
72 template< typename Info, int Char, int... Chars >
73 struct keyword {
