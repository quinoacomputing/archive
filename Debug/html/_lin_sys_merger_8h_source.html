3  \file src/LinSys/LinSysMerger.h
4  \copyright 2012-2015, J. Bakosi, 2016-2017, Los Alamos National Security, LLC.
5  \brief Charm++ chare linear system merger group to solve a linear system
6  \details Charm++ chare linear system merger group used to collect and
7  assemble the left hand side matrix (lhs), the right hand side (rhs) vector,
8  and the solution (unknown) vector from individual worker (e.g., Carrier)
9  chares. Beside collection and assembly, the system is also solved. The
10  solution is outsourced to hypre, an MPI-only library. Once the solution is
11  available, the individual worker chares are updated with the new solution.
12 
13  In the basic configuration this class assembles and solves a single linear
14  system, whose rhs vector may change during time stepping. In a more advanced
15  configuration, an additional, auxiliary, linear system can also be solved.
16  The basic configuration is configured by instantiating this class using the
17  AuxSolver = AuxSolverNull template argument, while the one that solves the
18  auxiliary system is configured by an AuxSolver class that provides a number
19  of member functions. For possible auxiliary solver classes, see, e.g.,
20  src/Inciter/AuxSolver.h.
21 
22  When LinSysMerger is configured to solve an auxiliary solution beside its
23  primary solution, the class behind the AuxSolver template argument, must
24  have static member functions only, which may provide the necessary
25  functionality to collect, assemble, solve, and update an auxiliary linear
26  system. The requirements on the auxiliary linear system are: (1) the left
27  hand side matrix must be diagonal, (2) the right hand side vector is
28  (optionally) a combination of the primary right hand side vector and another
29  vector, assembled separately (and overlapped with the primary system). This
30  enables configuring the auxiliary system to be, e.g., the low order solution
31  as required by the flux-corrected transport algorithm used for transport
32  equations in inciter. However, it also enables entirely removing the
33  auxiliary solution from LinSysMerger, via inciter::AuxSolverNull. Tthe
34  Charm++ SDAG logic does not change depending on whether an auxiliary
35  solution is performed or not. The composition is done at compile time,
36  depending on how LinSysMerger is instantiated.
37 
38  The implementation uses the Charm++ runtime system and is fully
39  asynchronous, overlapping computation and communication. The algorithm
40  utilizes the structured dagger (SDAG) Charm++ functionality. The high-level
41  overview of the algorithm structure and how it interfaces with Charm++ is
42  discussed in the Charm++ interface file src/LinSys/linsysmerger.ci. Note
43  that the SDAG logic is the same regardless whether an auxiliary solution is
44  performed.

340  //! Receive lower and upper global node IDs all PEs will operate on
341  //! \param[in] p PE whose bounds being received
342  //! \param[in] lower Lower index of the global rows on my PE
343  //! \param[in] upper Upper index of the global rows on my PE
344  void bounds( int p, std::size_t lower, std::size_t upper ) {
345  Assert( lower < upper, "Lower bound must be lower than the upper bound: "
346  "(" + std::to_string(lower) + "..." + std::to_string(upper) +
347  ") sent by PE " + std::to_string(p) );
348  // Store our bounds
349  if (p == CkMyPe()) {
350  m_lower = lower;
351  m_upper = upper;
352  }
353  // Store inverse of PE-division map stored on all PE
354  m_div[ {lower,upper} ] = p;
355  // If we have all PEs' bounds, signal the runtime system to continue
356  if (m_div.size() == static_cast<std::size_t>(CkNumPes())) {
357  // Create my PE's lhs matrix distributed across all PEs
358  m_A.create( m_lower*m_ncomp, m_upper*m_ncomp );
359  // Create my PE's rhs and unknown vectors distributed across all PEs
360  m_b.create( m_lower*m_ncomp, m_upper*m_ncomp );
361  m_x.create( m_lower*m_ncomp, m_upper*m_ncomp );
362  // Create linear solver
363  m_solver.create();
364  // Signal back to host that setup of workers can start
365  signal2host_coord( m_host );
366  }

402  //! \note This function does not have to be declared as a Charm++ entry

405  // Collect global ids of workers on my PE
406  m_myworker.push_back( fromch );
407  // Store rows owned and pack those to be exported, also build import map

410  for (auto gid : row) {
411  if (gid >= m_lower && gid < m_upper) { // if own
412  m_rowimport[ fromch ].push_back( gid );

417  m_nperow += exp.size();

772  //! Return processing element for global mesh row id
773  //! \param[in] gid Global mesh point (matrix or vector row) id
774  //! \details First we attempt to the point index in the cache. If that
775  //! fails, we resort to a linear search across the division map. Once the
776  //! PE is found, we store it in the cache, so next time the search is
777  //! quicker. This procedure must find the PE for the id.
778  //! \return PE that owns global row id
779  int pe( std::size_t gid ) {
780  int p = -1;
781  auto it = m_pe.find( gid );
782  if (it != end(m_pe))
783  p = it->second;
784  else
785  for (const auto& d : m_div)
786  if (gid >= d.first.first && gid < d.first.second)
787  p = m_pe[ gid ] = d.second;
788  Assert( p >= 0, "PE not found for node id " + std::to_string(gid) );
789  Assert( p < CkNumPes(), "Assigning to nonexistent PE" );
790  return p;
791  }

967  //! Set boundary conditions on the left-hand side matrix
968  //! \details Setting boundary conditions on the left-hand side matrix is
969  //! done by zeroing all nonzero entries in rows where BCs are prescribed
970  //! and putting in 1.0 for the diagonal.
971  void lhsbc() {
972  Assert( lhscomplete(),
973  "Nonzero values of distributed matrix on PE " +
974  std::to_string( CkMyPe() ) + " is incomplete: cannot set BCs" );
975 
976  // Set Dirichlet BCs on the lhs matrix. Loop through all BCs and if a BC
977  // is prescribed on a row we own, find that row (r) and in that row the
978  // position of the diagonal entry (diag) in the LHS matrix. Then for all
979  // scalar components the system of system of PDEs we intagrate, query the
980  // entry in the BC data structure to see if we need to set BC for the
981  // given component and set the off-diagonals to zero while put 1.0 into
982  // the diagonal.
983  for (const auto& n : m_bc) {
984  if (n.first >= m_lower && n.first < m_upper) {
985  auto& r = tk::ref_find( m_lhs, n.first );
986  auto& diag = tk::ref_find( r, n.first );
987  for (std::size_t i=0; i<m_ncomp; ++i)
988  if (n.second[i].first) {
989  for (auto& c : r) c.second[i] = 0.0; // zero columns in BC row
990  diag[i] = 1.0; // put 1.0 in diagonal of BC row
991  }
992  }
993  }
994  lhsbc_complete();
995  }
