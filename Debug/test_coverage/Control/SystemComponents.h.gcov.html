       7                 :            :     e.g. multiple equation sets of a physical model or a set of (stochastic
       8                 :            :     ordinary or partial differential) equations of different kinds.
       9                 :            : 
      10                 :            :     _Problem:_ We are given a type that is a tk::tuple::tagged_tuple that
      11                 :            :     contains an arbitrary number of std::vectors of integers. The number of
      12                 :            :     vectors are fixed at compile-time (accessed via tags) but their (differing)
      13                 :            :     length is only known at run-time (after parsing user input). What we need
      14                 :            :     are functions that operate on this data structure and return, e.g., the
      15                 :            :     total number of components in the whole system or the offset in the whole
      16                 :            :     data structure for a given tag. The functions should thus be able to operate
      17                 :            :     on a list of types, i.e., a double for loop over all tags and associated
      18                 :            :     vectors - one at compile-time and the other one at run-time.
      19                 :            : 
      20                 :            :     _Example:_ An example is to define storage for systems of stochastic
      21                 :            :     differential equations as in walker::ctr::ncomps. In
      22                 :            :     [walker](walker.html) various types of stochastic differential
      23                 :            :     equations are implemented and numerically integrated in time in order to
      24                 :            :     compute and learn about their statistical behavior. Since different types of
      25                 :            :     systems can be parameterized differently, there can be several systems of
      26                 :            :     the same type (the number is user-defined so only known at run-time). For
      27                 :            :     example, it is possible to numerically integrate two systems of Dirichlet
      28                 :            :     equations, see DiffEq/Dirichlet.h, one with 4, the other one with 5 scalar
      29                 :            :     variables, parameterized differently, and estimate their arbitrary coupled
      30                 :            :     statistics and joint PDFs. This requires that each type of system
      31                 :            :     has a vector of integers storing the number of scalar variables.
      32                 :            : 
      33                 :            :     _Solution:_ Looping through elements of a tuple is done via [Boost]
      34                 :            :     (http://www.boost.org)'s [MetaProgramming Library (MPL)]
      35                 :            :     (http://www.boost.org/doc/libs/release/libs/mpl)'s [for_each]
      36                 :            :     (http://www.boost.org/doc/libs/release/libs/mpl/doc/refmanual/for-each.html).
      37                 :            :     Such operations on types happen at compile-time, i.e., the code runs inside
      38                 :            :     the compiler and only its result gets compiled into code to be run at
      39                 :            :     run-time. Advantages are abstraction and generic code that is independent
      40                 :            :     of the size and order of the tags in the tuple (and the associated vectors).
      41                 :            :     Though it is not of primary concern for this data structure, this solution
      42                 :            :     also generates efficient code, since part of the algorithm (for computing
      43                 :            :     the offset, the total number of components, etc.) runs inside the compiler.
      44                 :            :     All of this is kept generic, so it does not need to be changed when a new
      45                 :            :     pair of tag and system of equations are added to the underlying tuple.
      46                 :            :     Thus, _the main advantage is that changing the order of the vectors in the
      47                 :            :     tuple or adding new ones at arbitrary locations will not require change to
      48                 :            :     client-code._
      49                 :            : 
      50                 :            :     _An alternative approach:_ Of course this could have been simply done with a
      51                 :            :     purely run-time vector of vector of integers. However, that would not have
      52                 :            :     allowed a tag-based access to the different systems of equations. (Sure, one
      53                 :            :     can define an enum for equation indexing, but that just adds code to what is
      54                 :            :     already need to be changed if a change happens to the underlying data
      55                 :            :     structure.) As a consequence, a component in a vector of vector would have
      56                 :            :     had to be accessed as something like, ncomps[2][3], which is more
      57                 :            :     error-prone to read than ncomps< tag::dirichlet >( 3 ). Furthermore, that
      58                 :            :     design would have resulted in double-loops at run-time, instead of letting
      59                 :            :     the compiler loop over the types at compile-time and do the run-time loops
      60                 :            :     only for what is only known at run-time. Additionally, and most importantly,
      61                 :            :     _any time a new system is introduced (or the order is changed), all
      62                 :            :     client-code would have to be changed._
      63                 :            : 
      64                 :            :     For example client-code, see walker::Dirichlet::Dirichlet() in
