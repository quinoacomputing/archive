5  \brief Charm++ chare partitioner group used to perform mesh partitioning
6  \details Charm++ chare partitioner group used to perform mesh partitioning.
7 
8  The implementation uses the Charm++ runtime system and is fully
9  asynchronous, overlapping computation, communication as well as I/O. The
10  algorithm utilizes the structured dagger (SDAG) Charm++ functionality. The
11  high-level overview of the algorithm structure and how it interfaces with
12  Charm++ is discussed in the Charm++ interface file
13  src/Inciter/partitioner.ci.
14 
15  #### Call graph ####
16  The following is a directed acyclic graph (DAG) that outlines the
17  asynchronous algorithm implemented in this class The detailed discussion of
18  the algorithm is given in the Charm++ interface file partitioner.ci,
19  which also repeats the graph below using ASCII graphics. On the DAG orange
20  fills denote global synchronization points, orange frames with white fill
21  are partial synchronization points that overlap with other tasks, and dashed
22  lines are potential shortcuts that allow jumping over some of the task-graph
23  under some circumstances. See the detailed discussion in partitioner.ci.
24  \dot
