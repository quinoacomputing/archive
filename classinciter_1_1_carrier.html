"Contiguous-row-id" here means that the numbering of the mesh nodes (which corresponds to rows in the linear system) are (approximately) contiguous (as much as this can be done with an unstructured mesh) as the problem is distirbuted across PEs, held by LinSysMerger objects. This ordering is in start contrast with "as-in-file" ordering, which is the ordering of the mesh nodes as it is stored in the file from which the mesh is read in. The as-in-file ordering is highly non-contiguous across the distributed problem.
