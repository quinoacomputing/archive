2 /*!
3  \file src/Inciter/CG.h
4  \copyright 2012-2015, J. Bakosi, 2016-2017, Los Alamos National Security, LLC.
5  \brief CG advances a system of PDEs with the continuous Galerkin scheme
6  \details CG advances a system of partial differential equations (PDEs) using
7  continuous Galerkin (CG) finite element (FE) spatial discretization (using
8  linear shapefunctions on tetrahedron elements) combined with a time stepping
9  scheme that is equivalent to the Lax-Wendroff (LW) scheme within the
10  unstructured-mesh FE context and treats discontinuities with flux-corrected
11  transport (FCT).
12 
13  There are a potentially large number of CG Charm++ chares created by
14  Transporter. Each CG gets a chunk of the full load (part of the mesh)
15  and does the same: initializes and advances a number of PDE systems in time.
16 
17  The implementation uses the Charm++ runtime system and is fully
18  asynchronous, overlapping computation and communication. The algorithm
19  utilizes the structured dagger (SDAG) Charm++ functionality. The high-level
20  overview of the algorithm structure and how it interfaces with Charm++ is
21  discussed in the Charm++ interface file src/Inciter/cg.ci.
22 
23  #### Call graph ####
24  The following is a directed acyclic graph (DAG) that outlines the
25  asynchronous algorithm implemented in this class The detailed discussion of
26  the algorithm is given in the Charm++ interface file transporter.ci, which
27  also repeats the graph below using ASCII graphics. On the DAG orange
28  fills denote global synchronization points that contain or eventually lead
29  to global reductions. Dashed lines are potential shortcuts that allow
30  jumping over some of the task-graph under some circumstances or optional
31  code paths (taken, e.g., only in DEBUG mode). See the detailed discussion in
32  cg.ci.
