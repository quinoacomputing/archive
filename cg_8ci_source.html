50  // Interpretation of the above DAG

52  // Control flow is from left to right and top to bottom.
53  //
54  // Only in DEBUG mode, wait4ver(): When the low and high order solutions
55  // have been updated, and the own portion of the antidiffusive element
56  // contributions (AEC) have been computed, we optionally (inly in DEBUG
57  // mode) verify that the AECs assembled to nodes equal to the difference
58  // between the high and low order increments. See also
59  // FluxCorrector::aec() and FluxCorrector::verify(). Note that this
60  // verification is only done if there is only a single CG chare as
61  // the correct verification would entail communication for multiple
62  // chares. Note also that the DAG conditions below for wait4ver() require
63  // the own contributions to the AEC and ALW instead of requiring the low
64  // and high order solutions and the own AEC. Requiring the former set (as
65  // implemented) is stronger than the latter, because the former can only
66  // be satisfied when the latter is also satisfied. The former is
67  // implemented because that avoids introducing two more SDAG messages,
68  // e.g., lo_complete() and hi_complete(), and that would require 3
69  // conditions instead of two, leading to simpler code.
70  //
71  // Computing limited antidiffusive element contributions, wait4fct():
72  // Computing and communicating contributions on chare-boundaries of
73  // antidiffusive element contributions (AEC), and the maximum and minimum
74  // unknowns of elements surrounding nodes (ALW) are overlapped. See
75  // aec(), comaec(), alw(), comalw(). When all are complete on a single
76  // chare, computing the limited antidiffusive element contributions can
77  // start.
78  //
79  // Applying the limited antidiffusive element contributions, wait4app():
80  // Computing the limited antidiffusive element contributions assembled and
81  // applied to mesh nodes involves own and contributions on the
82  // chare-boundary nodes. When both are done on a single chare, applying
83  // the new limited AECs can be done.
84  //
85  // Note that the none of wait4ver(), wait4fct(), and wait4app(), nor its
86  // underlying computations and communications involve global reductions or
87  // broadcasts: the entire FCT algorithm runs completely independent on
88  // each CG chare, and exercising task parallelism within.
89 
90  #ifndef NDEBUG

92  when ownaec_complete(), ownalw_complete() serial "ver" { verify(); } };
