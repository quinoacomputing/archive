10  The implementation uses the Charm++ runtime system and is fully
11  asynchronous, overlapping computation and communication. The algorithm
12  utilizes the structured dagger (SDAG) Charm++ functionality. The high-level
13  overview of the algorithm structure and how it interfaces with Charm++ is
14  discussed in the Charm++ interface file src/Inciter/transporter.ci.
15 
16  #### Call graph ####
17  The following is a directed acyclic graph (DAG) that outlines the
18  asynchronous algorithm implemented in this class The detailed discussion of
19  the algorithm is given in the Charm++ interface file transporter.ci, which
20  also repeats the graph below using ASCII graphics. On the DAG orange
21  fills denote global synchronization points that contain or eventually lead
22  to global reductions. Dashed lines are potential shortcuts that allow
23  jumping over some of the task-graph under some circumstances or optional
24  code paths (taken, e.g., only in DEBUG mode). See the detailed discussion in
25  transporter.ci.
26  \dot
27  digraph "Transporter SDAG" {

29  node [shape=record, fontname=Helvetica, fontsize=10];
