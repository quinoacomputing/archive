void NodeDiagnostics::registerReducers ( )
static

Configure Charm++ custom reduction types initiated from this class.

This routine is supposed to be called from a Charm++ nodeinit routine. Since the runtime system executes nodeinit routines exactly once on every logical node early on in the Charm++ init sequence, they must be static as they are called without an object. See also: Section "Initializations at Program Startup" at in the Charm++ manual http://charm.cs.illinois.edu/manuals/html/charm++/manual.html.
