54 DiffEqStack::DiffEqStack() : m_factory(), m_eqTypes()
55 // *****************************************************************************
56 // Constructor: register all differential equations into factory
57 //! \details This constructor consists of several blocks, each registering a
58 //! potentially large number of entries in the differential equation factory,
59 //! m_factory, which is of type walker::DiffEqFactory, a std::map. At this
60 //! time, each type of differential equation can be configured to use a unique
61 //! _initialization policy_ and a unique _coefficients policy_. (More types
62 //! of policies will most likely come in the future.) The policy classes are
63 //! template arguments to the differential equation classes and influence
64 //! their behavior in a different way, abstracting away certain functions,
65 //! e.g., how to set initial conditions and how to update their coefficients
66 //! during time integration. For more information on policy-based design, see
67 //! http://en.wikipedia.org/wiki/Policy-based_design. This abstraction allows
68 //! [separation of concerns](http://en.wikipedia.org/wiki/Separation_of_concerns).
69 //!
70 //! Since the functionality of the policies are orthogonal to each other,
71 //! i.e., they do not depend on each other or their host (the differential
72 //! equation class), a Cartesian product of combinations are possible,
73 //! depending on which policies are selected. _This constructor registers all
74 //! possible combinations of policies for all available differential
75 //! equations._ By _register_, we mean, an entry is recorded in an associative
76 //! container, a std::map, that associates a lightweight key of type
77 //! walker::ctr::DiffEqKey, consisting of only an enum for each policy type,
78 //! to an std::function object that holds the constructor bound to its
79 //! arguments corresponding to a particular differential equation + policies
80 //! combination. Note that registering these entries in the map does not
81 //! invoke the constructors. The mapped value simply stores how the
82 //! constructors should be invoked at a later time. At some point later,
83 //! based on user input, we then instantiate only the differential equations
84 //! (and only those configurations) that are requested by the user.
85 //!
86 //! Since all differential equation types (registered in the factory)
87 //! "inherit" from a common "base", client-code is unform and generic, and
88 //! thus immune to changes in the inner workings of the particular
89 //! differential equations as long as they fullfill certain concepts, i.e.,
90 //! implement certain member functinos, enforced by the _common base_, DiffEq.
91 //! The words "inherit and "base" are quoted here, because the common base
92 //! does not use inheritance in the normal OOP sense and does not use
93 //! reference semantics, i.e., pointers, visible to client-code either. The
94 //! relationship is more of a _models a_-type, which simplifies client-code
95 //! and allows for the benfits of runtime inheritance with value-semantics
96 //! which is less error prone and easier to read. See more about the
97 //! _models-a_ relationship and its implementation in DiffEq/DiffEq.h.
98 //!
99 //! The design discussed above allows the registration, instantiation, and
100 //! use of the differential equations to be generic, which eliminates a lot of
101 //! boiler-plate code and makes client-code uniform.
102 //!
103 //! _Details of registration using mpl::cartesian_product:_
104 //!
105 //! The template argument to mpl::cartesian_product requires a sequence of
106 //! sequences of types. We use vector of vectors of types, listing all
107 //! possible policies. The constructor argument to mpl::cartesian_product is a
108 //! functor that is to be applied to all combinations. mpl::cartesian_product
109 //! will then create all possible combinations of these types and call the
110 //! user-supplied functor with each type of the created sequence as a template
111 //! parameter. The user-supplied functor here is registerDiffEq, which, i.e.,
112 //! its constructor call, needs a single template argument, a class templated
113 //! on policy classes. This is the differential equation class to be
114 //! configured by selecting policies and to be registered. The arguments to
115 //! registerDiffEq's constructor are the factory, the enum denoting the
116 //! differential equation type, and a reference to a variable of type
117 //! std::set< ctr::DiffEqType >, which is only used internally to DiffEqStack
118 //! for counting up the number of unique differential equation types
119 //! registered, used for diagnostics purposes.
120 // *****************************************************************************

228 std::vector< walker::DiffEq >
229 DiffEqStack::selected() const
230 // *****************************************************************************
231 // Instantiate all selected differential equations
232 //! \return std::vector of instantiated differential equation objects
233 // *****************************************************************************
234 {
235  std::map< ctr::DiffEqType, ncomp_t > cnt; // count DiffEqs per type
236  std::vector< DiffEq > diffeqs; // will store instantiated DiffEqs
237 
238  for (const auto& d : g_inputdeck.get< tag::selected, tag::diffeq >()) {
239  if (d == ctr::DiffEqType::DIRICHLET)
240  diffeqs.push_back( createDiffEq< tag::dirichlet >( d, cnt ) );
241  else if (d == ctr::DiffEqType::GENDIR)
242  diffeqs.push_back( createDiffEq< tag::gendir >( d, cnt ) );
243  else if (d == ctr::DiffEqType::WRIGHTFISHER)
244  diffeqs.push_back( createDiffEq< tag::wrightfisher >( d, cnt ) );
245  else if (d == ctr::DiffEqType::OU)
246  diffeqs.push_back( createDiffEq< tag::ou >( d, cnt ) );
247  else if (d == ctr::DiffEqType::DIAG_OU)
248  diffeqs.push_back( createDiffEq< tag::diagou >( d, cnt ) );
249  else if (d == ctr::DiffEqType::BETA)
250  diffeqs.push_back( createDiffEq< tag::beta >( d, cnt ) );
251  else if (d == ctr::DiffEqType::NUMFRACBETA)
252  diffeqs.push_back( createDiffEq< tag::numfracbeta >( d, cnt ) );
253  else if (d == ctr::DiffEqType::MASSFRACBETA)
254  diffeqs.push_back( createDiffEq< tag::massfracbeta >( d, cnt ) );
255  else if (d == ctr::DiffEqType::MIXNUMFRACBETA)
256  diffeqs.push_back( createDiffEq< tag::mixnumfracbeta >( d, cnt ) );
257  else if (d == ctr::DiffEqType::MIXMASSFRACBETA)
258  diffeqs.push_back( createDiffEq< tag::mixmassfracbeta >( d, cnt ) );
259  else if (d == ctr::DiffEqType::SKEWNORMAL)
260  diffeqs.push_back( createDiffEq< tag::skewnormal >( d, cnt ) );
261  else if (d == ctr::DiffEqType::GAMMA)
262  diffeqs.push_back( createDiffEq< tag::gamma >( d, cnt ) );
263  else Throw( "Can't find selected DiffEq" );
264  }
265 
266  return diffeqs;
