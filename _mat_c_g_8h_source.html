2 /*!
3  \file src/Inciter/MatCG.h
4  \copyright 2012-2015, J. Bakosi, 2016-2018, Los Alamos National Security, LLC.
5  \brief MatCG for a PDE system with continuous Galerkin with a matrix
6  \details MatCG advances a system of partial differential equations (PDEs)
7  using continuous Galerkin (CG) finite element (FE) spatial discretization
8  (using linear shapefunctions on tetrahedron elements) combined with a time
9  stepping scheme that is equivalent to the Lax-Wendroff (LW) scheme within
10  the unstructured-mesh FE context and treats discontinuities with
11  flux-corrected transport (FCT). The left-hand side (consistent-mass) matrix
12  is stored in a compressed sparse row (CSR) storage and thus this scheme uses
13  a matrix-based linear solver.
14 
15  There are a potentially large number of MatCG Charm++ chares created by
16  Transporter. Each MatCG gets a chunk of the full load (part of the mesh)
17  and does the same: initializes and advances a number of PDE systems in time.
18 
19  The implementation uses the Charm++ runtime system and is fully
20  asynchronous, overlapping computation and communication. The algorithm
21  utilizes the structured dagger (SDAG) Charm++ functionality. The high-level
22  overview of the algorithm structure and how it interfaces with Charm++ is
23  discussed in the Charm++ interface file src/Inciter/matcg.ci.
24 */
