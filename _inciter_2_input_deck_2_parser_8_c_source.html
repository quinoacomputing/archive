48  // Reset parser's output stream to that of print's. This is so that mild
49  // warnings emitted during parsing can be output using the pretty printer.
50  // Usually, errors and warnings are simply accumulated during parsing and
51  // printed during diagnostics after the parser has finished. Howver, in some
52  // special cases we can provide a more user-friendly message right during
53  // parsing since there is more information available to construct a more
54  // sensible message. This is done in e.g., tk::grm::store_option. Resetting
55  // the global g_print, to that of passed in as the constructor argument allows
56  // not to have to create a new pretty printer, but use the existing one.
57  tk::grm::g_print.reset( print.save() );
